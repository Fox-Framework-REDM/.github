# 🦊 Fox Framework

A modern RedM framework built from a port of the Ox ecosystem and adapted.

Fox Framework aims to provide a solid, scalable, and developer-friendly foundation while preserving the performance, modularity, and server-authoritative philosophy that made the Ox ecosystem popular.

---

## Overview

Fox Framework is designed as a complete RedM framework focused on:

- Performance
- Scalability
- Security
- Modularity
- Developer Experience
- Long-Term Maintainability

The project is being developed specifically for RedM and takes advantage of Red Dead Redemption 2 systems such as horses, wagons, camps, and native game mechanics.

---

## Core Features

### Character System

- Persistent character management
- Character metadata
- Identity system
- Character appearance support

### Multi Character

- Multiple character slots

- Expandable slot support

### Groups & Permissions

- Native group system
- Role-based permissions

- Client callbacks
- Server callbacks

- Secure request handling

### Inventory Integration

- Ox Inventory compatibility
- Metadata support
- Extensible item system

### Session Management

- Player session handling

- State management

### Database Layer

- Optimized database queries

### NUI Framework

- React support
- TypeScript support

## Project Philosophy

### Performance First

Systems should minimize:

- Database queries
- Network traffic
- Resource usage
- Unnecessary loops

### Server Authority

Critical gameplay systems should always be validated server-side whenever possible.

### Modularity

Resources should remain independent and easy to replace, extend, or integrate.

### Developer Experience

Fox Framework focuses on:

- Clean APIs
- Consistent exports
- Predictable behavior
- Maintainable code
